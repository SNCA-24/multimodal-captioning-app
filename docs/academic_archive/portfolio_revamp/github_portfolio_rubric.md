  
**Master Repo Scoring Rubric**

**Scoring criteria**

Score each repo on **1 to 5** for these:

1. **Role relevance**  
   How useful is it for **MLE / DE / DS** hiring?  
2. **Engineering signal**  
   Does it show architecture, pipelines, systems, APIs, reproducibility, deployment thinking?  
3. **ML depth**  
   Does it show real modeling, evaluation, tradeoffs, experimentation?  
4. **Differentiation**  
   Does it stand out from generic student projects?  
5. **Resume/story value**  
   Will it strengthen your overall profile narrative?  
6. **Polish effort**  
   How easy is it to convert into a strong showcase repo?  
   Here use reverse logic:  
   * **5 \= easier to polish**  
   * 3 \= medium effort to polish  
   * **1 \= very hard to polish**

**Final weighted score**

Use this weight:

* Role relevance × **25**  
* Engineering signal × **20**  
* ML depth × **15**  
* Differentiation × **15**  
* Resume/story value × **15**  
* Polish effort × **10**   
  Total \= **100**
